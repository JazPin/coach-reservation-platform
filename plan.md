## 1. 產品定位

### 一句話定義

> **專為台灣自由健身教練設計的輕量行政後台——讓教練把時間花在教學，而不是 Excel 和 LINE 追款。**

### 目標用戶

**主要：獨立接案教練（自由工作室型）**

- 不依附健身房、自帶場地或租場
- 同時管理 15–40 位學員
- 收款靠銀行轉帳或現金，排課靠 LINE + Excel
- 每週花 3–5 小時在行政雜事
- 月收入 NT$30,000–200,000

**次要：場館內兼差教練**

- 有場館系統協助，但私下自招學員時面臨同樣痛點
- 付費意願中等，適合作為第二階段擴散對象

### 核心承諾

「一個自由教練管理 30 位學員，不再需要 Excel 和手動對帳。」

## 2. 市場與痛點驗證

### 市場規模

| 指標 | 數據 |
|---|---|
| 台灣持照私人教練估計人數 | 30,000+ |
| 一對一每堂收費 | NT$1,500–3,500 |
| 獨立教練平均管理學員數 | 20–40 人 |
| 學員 3 個月課程平均消費 | NT$45,000+ |

### 三大已驗證痛點

以下痛點均來自 PTT MuscleBeach、Dcard 健身版、Medium 教練文章的真實用戶聲音：

#### 痛點 1：課程與財務管理全靠手工

> 「是否熟悉或有一套自己的工作流程——銷課報表、學生紀錄、課程合約等等」
> — 時尚肌肉，自由教練轉職分析文

> 「自由教練就是一人公司，維持公司經營的最大基礎就是財務。每個月要幫自己製作財務報表。」
> — Medium，自由教練年收入分析

**結論：** 教練自己知道需要「銷課報表 + 學生紀錄」，但目前都是手工或 Excel。

#### 痛點 2：爽約與到課率是顯著經營痛點

> 「另一個常被忽視的問題是預約學生的到課情況記錄。」
> — Medium，自由教練第一課：提升客戶到課率

> 「說要排課，都是很零碎的時間，他說可以才可以」
> — PTT MuscleBeach 留言

**結論：** 有人專門寫文章談「如何提升到課率」，代表這個問題夠普遍、夠痛。

#### 痛點 3：收費現金／轉帳，沒有正式金流紀錄

> 「收費都是現金支付。當銀行帳號裡沒有任何關於薪資的帳目往來時，你要買房或申請信貸就知道了——你之於銀行是信用空白的人。」
> — The News Lens，自由教練深度分析

**結論：** 現金收款不只行政麻煩，還有長期財務隱患（無法貸款）。

### 競品缺口分析

| 工具 | 定位 | 缺口 |
|---|---|---|
| Trainge | 課程媒合平台 | 非教練日常管理工具 |
| JoiiCoach | 線上訓練計畫 | 適合線上教練，非現場一對一 |
| BookFast POS | 健身房場館系統 | 太貴太重，個人教練用不起 |
| SimplyMeet | 通用預約工具 | 無課堂扣點、學員進度、訓練紀錄 |
| 健身遊牧 | 教練媒合接案 | 非日常管理工具 |

**結論：** 無任何工具專為「獨立接案教練的日常運營管理」設計。

## 3. 金流架構設計

### 推薦架構：教練自有綠界帳號（選項 B）

**金流路徑：**

```
學員付款 → 綠界（教練自己的帳號）→ 教練銀行帳戶
                    ↓
              Webhook 事件通知
                    ↓
              你的平台：自動更新扣點餘額
```

**你的公司完全不碰錢。**

### 為什麼選這個架構

**法規面：** 若平台代收代付，需登記「第三方支付服務業」並提供信託專戶。早期一人公司負擔重。教練自有帳號架構完全規避此問題。

**工程面：** 最簡單。平台只需讀取 Webhook 通知，更新資料庫中的「堂數」數字。

**商業面：** 你的收入來源是教練月費（$499/月），與學員課程金流完全分離。兩條金流獨立，法律最乾淨。

### 教練端設定流程（一次性）

1. 教練至綠界官網申請個人戶帳號（免費，線上填單）
2. 取得 Hash Key、Hash IV、Merchant ID
3. 貼入平台設定頁面，完成串接

### 平台側工程重點

平台使用教練的 API 金鑰，呼叫綠界 API 產生付款頁面，設定 `ReturnURL` 作為 Webhook 端點。收到 Webhook 後，**必須驗證 `CheckMacValue` 簽章**（防止偽造付款通知），驗證通過後才更新學員堂數。

```
Webhook 驗證流程：
1. 接收 POST 請求
2. 取出 CheckMacValue
3. 用教練的 Hash Key + Hash IV 重新計算
4. 比對一致 → 更新扣點餘額
5. 比對不一致 → 拒絕，記錄異常
```

### 你的公司收入結構

| 收入來源 | 金額 | 說明 |
|---|---|---|
| 教練平台月費 | NT$499/月/教練 | 走你自己申請的綠界帳號 |
| 學員課程費 | 不碰 | 直接進教練帳戶 |
| 交易抽成 | 無 | 初期不做，避免法規複雜度 |

## 5. 技術架構

### 技術選型（一人可執行）

| 層級 | 選擇 | 理由 |
|---|---|---|
| 前端（教練後台） | Next.js + Tailwind CSS | 全端整合，部署簡單 |
| 前端（學員預約頁） | 純 HTML 靜態頁 | 無需學員登入，速度快 |
| 後端 | Next.js API Routes | 與前端共用同一 repo |
| 資料庫 | Supabase (PostgreSQL) | Auth + Real-time 內建，免費額度夠用 |
| 金流 | 綠界 ECPay API | 個人可申請，文件完整，台灣最普及 |
| LINE 提醒 | Email（V1 過渡）→ LINE Messaging API（V2 評估）| LINE Notify 已宣布 deprecate，V1 改以 Email 替代；V2 再評估 LINE Messaging API |
| AI 功能 | Anthropic API (claude-sonnet) | 課後日誌、續課草稿 |
| 部署 | Vercel | 免費額度夠用到 50 位教練 |

### 初期月費成本

| 服務 | 費用 |
|---|---|
| Supabase | $0（免費額度） |
| Vercel | $0（免費額度） |
| LINE Notify | $0 |
| Anthropic API | 依用量，初期 < $500/月 |
| 綠界平台月費 | $0（按交易手續費計）|
| **總固定成本** | **幾乎為零** |

### 資料庫核心 Schema

```
coaches
  - id, email, name, line_notify_token, ecpay_merchant_id, ecpay_hash_key, ecpay_hash_iv

students
  - id, coach_id, name, phone, email, goal, notes, created_at

session_packages
  - id, student_id, coach_id, total_sessions, remaining_sessions, price, paid_at

appointments
  - id, student_id, coach_id, scheduled_at, status (scheduled/completed/no_show)
  - reminded_48h (bool), reminded_24h (bool)

session_logs
  - id, appointment_id, notes, weight_kg, body_fat_pct, logged_at
```

## 6. GTM 策略

### 目標：3 個月 50 位付費教練

#### 月 1：找到 5 位種子教練

目標不是賣產品，是「一起解決問題」。

**找教練的方法：**
- 搜尋 IG `#台灣健身教練`、`#自由教練`，找粉絲數 1,000–10,000 的中型教練
- 在 Dcard 健身版留言有幫助的回覆，建立存在感
- 健身遊牧、Trainge 教練社群發文（求助型，非廣告）

**開場白：**
> 「我在做一個專給自由教練用的學員管理工具，目前免費，只需要你每週給我 10 分鐘的反饋。你願意試試看嗎？」

**驗證指標：** 這 5 位教練每週用幾次？最常用哪個功能？

#### 月 2：讓種子教練說話

如果教練真的每天用，請他們錄一支 30 秒的 Reels：

> 「以前管 30 個學員，用 LINE 筆記 + Excel，每週花 3 小時在行政。現在只要幾分鐘…」

V2 在月 2 末推出：金流上線，開始收 $499/月。種子教練前 6 個月免費。

#### 月 3：進入社群，批量擴散

**目標社群：**
- Facebook：「健身教練經驗分享」、「台灣私人教練」等社群（總計 2–5 萬人）
- Dcard 健身版
- PTT MuscleBeach

**溝通策略：** 不打廣告，發心得文。

> 「我管 35 個學員，以前靠 Excel 快崩潰，現在用這個工具，分享一下我的工作流程。」

### 3 個月里程碑

## 7. 商業模式與定價

### 定價分層

| 方案 | 月費 | 學員上限 | 功能 | 策略目的 |
|---|---|---|---|---|
| **免費** | $0 | 5 位 | 基本扣點 + 排課，無金流 | 零摩擦試用，建立習慣 |
| **專業** | NT$499 | 無限 | 全功能：金流 + LINE 提醒 + AI | 主要收入來源 |
| **工作室**（未來）| NT$1,499 | 多教練帳號 | 統一後台，業績報表 | 擴展到小型工作室 |

### 定價邏輯

一位教練管理 30 位學員，月收 NT$15,000–60,000（以每月 2 堂計）：

- 平台費 NT$499 = 月收入的 **0.8–3.3%**
- 對應節省的時間（每週 3–5 小時，換算時薪最少值 NT$5,000–10,000/月）
- 投資報酬率極高，接受度極高

|---|---|---|
| M3 | 50 | NT$24,950 |

## 9. 風險評估

### 已知風險與對策

| 風險 | 等級 | 對策 |
|---|---|---|
| LINE Notify 通知被忽略 | 中 | V2 升級 LINE Messaging API；確保訊息內容有足夠的信息量（時間、學員姓名、剩餘堂數） |
| 教練習慣 Excel，改變有阻力 | 高 | 把 Excel 匯入做成第一步，讓遷移幾乎無痛；種子教練共創讓他們有歸屬感 |
| 競品跟進 | 中 | 護城河靠 AI 課後日誌（通用工具很難做好）和教練社群黏性 |
| 市場天花板 | 低 | 台灣獨立教練約 1–3 萬人，轉換率 5% = 500–1,500 付費用戶；月收上限 NT$75–75 萬。夠撐一人公司，但不是大市場。如要擴展，後期考慮攻香港、新加坡中文市場 |
| 綠界串接複雜度 | 中 | Webhook 驗證是關鍵，約 30–50 行程式碼。綠界文件完整，有沙箱測試環境 |

### 不該做的事（明確邊界）

- ❌ 不做代收代付（避免法規複雜度）
- ❌ 不做學員 App（增加維護負擔，教練側工具才是核心）
- ❌ 不做健身房管理系統（定位是個人教練，不是企業）
- ❌ 不一次做太多功能（V1 只做扣點 + 排課，驗證後再加）
